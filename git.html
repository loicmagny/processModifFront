Utilisation de Git pour le projet

  1. Pour commencer, il est nécessaire d'avoir un compte Gitlab.
  2. Être invité sur le projet.
  3. Effectuer un git clone du projet.
  4. Créer une branche selon le format suivant -> prénom/tâche/fichier. Exemple: loic/traduction/rse.
  5. Une fois terminé, effectuer un git push puis un git merge de la branche vers la branche principale.
  6. Il faut résoudre les conflits s'il y en a. Les conflits sur theme.js ou theme.css peuvent être ignorés -> les fichiers sont régénérés à chaque modification css/js.
  7. Une fois le merge effectué et les conflits résolus, les fichiers peuvent être mis sur le serveur.

Commandes basiques de Git

  • git status permet d'afficher des informations telles que la branche en cours, les fichiers modifiés, etc. Cela est utile pour connaître l'état actuel du projet.

    Pour plus d'informations, consultez la documentation de git status

  • git add permet d'ajouter des fichiers à la préparation du commit.

    Utilisez git add . pour ajouter tous les fichiers modifiés.

    Pour plus d'informations, consultez la documentation de git add

    .
  • git commit permet de créer un nouveau commit avec les fichiers ajoutés.

    Par exemple, git commit -m "Ajout de nouvelles fonctionnalités".

    Pour plus d'informations, consultez la documentation de git commit

    .
  • git push permet d'envoyer les commits locaux vers le dépôt distant.

    Par exemple, git push origin master.

    Pour plus d'informations, consultez la documentation de git push

    .
  • git merge permet de fusionner une branche avec une autre branche.

    Par exemple, git merge feature-branch.

    Pour plus d'informations, consultez la documentation de git merge

    .
  • git stash permet de sauvegarder temporairement les modifications non validées.

    Par exemple, git stash save "Correction de bugs".

    Pour plus d'informations, consultez la documentation de git stash

    .
  • git pull permet de récupérer les derniers commits du dépôt distant et de les fusionner avec la branche locale.

    Par exemple, git pull origin master.

    Pour plus d'informations, consultez la documentation de git pull

    .
  • git fetch permet de récupérer les derniers commits du dépôt distant sans les fusionner avec la branche locale.

    Par exemple, git fetch origin.

    Pour plus d'informations, consultez la documentation de git fetch

    .
