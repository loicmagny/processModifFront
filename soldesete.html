Instructions pour modifier l'image pour les soldes d'été

Ouvrir VS Code

Ouvrir le dossier du projet situé dans: /Documents/Dev/www/SeviraKidsV3

Dans VS Code :

Ouvrir le dossier themes
Ouvrir le dossier classic-rocket-child
Ouvrir le dossier _dev
Ouvrir le dossier js puis le fichier responsive.js (double clic)

Dans le terminal :

Ouvrez un nouveau terminal

Tapez ou collez les commandes suivantes (appuyez sur entrée après chaque ligne) :

  • git status (Attendez la fin de l'exécution de la commande)

    Si la console affiche ceci, vous pouvez sauter l'étape du git pull

  • git pull (Attendez la fin de l'exécution de la commande)
  • cd themes/classic-rocket-child/_dev
  • npm run fulldebug

Dans responsive.js

Pour effectuer la modification:

  1. Retirez les // à la ligne 152
  2. Ajoutez des // à la ligne 150
  3. Sauvegardez la modification (Ctrl + S )
  4. Vérifiez dans le terminal que les fichiers sont compilés (attendez un peu si la compilation n'est pas terminée)
  5. compilation du JS terminée
  6. compilation du CSS terminée

Exporter les fichiers sur le serveur

Ouvrez FileZilla

Connectez-vous au serveur en utilisant les informations contenues dans log.txt (sur le bureau de la VM)

Une fois connecté:
Côté rouge: Ordinateur local
Côté bleu: Serveur
Explications
  1. Chemin absolu: L'endroit précis où l'on se trouve dans la machine
  2. Arborescence globale: L'ensemble des fichiers de la machine
  3. Arborescence détaillée: Le contenu d'un dossier

Avant de mettre les modifications en ligne sur la prod, on va les tester sur la preprod

Dans le dossier preprod.sevirakid.com:

Faire glisser les fichiers css et js situé dans themes/classic-rocket-child/assets de la machine locale vers le même dossier themes/classic-rocket-child/assets sur le serveur (cf image), confirmer le remplacement des fichiers

Mettre les modifications en prod

Une fois que les changements ont été testés et validés sur la preprod, on peut les mettre en ligne sur la prod.

Pour ce faire il suffit de répéter l'opération décrite au dessus mais dans le dossier seviraprod situé dans var/www/ sur le serveur

Astuce: Dans la barre contenant le chemin sur Filezilla, côté serveur, il est possible de remplacer preprod.sevirakids.com par seviraprod

Transformer var/www/preprod.sevirakids.com/themes/classic-rocket-child/assets/ en var/www/seviraprod/themes/classic-rocket-child/assets/

Permet de se déplacer instantanément dans le bon dossier sans avoir à parcourir toute l'arborescence afin de se déplacer

Etape finale ( errorvitale)

  1. Dans VS Code

  2. Si le terminal est fermé, en ouvrir un nouveau
  3. Copier/coller chaque commande et appuyer sur entrée après chacune d'entre elles :
    • git add .
    • git commit -m "modif img soldes" (ou un autre message entre les guillemets)
    • git push -u origin main

      • Pour faire le @, option droite + 9

    • git status -> Devrait afficher un message "Already up to date" (Déjà à jour)

      ça veut dire que tout s'est bien passé et que le dépôt distant est à jour

Pour Quitter

Quitter la machine virtuelle se fait de la même manière que quitter une fenêtre normale

Utiliser Éteindre dans le menu d'Ubuntu (en haut à droite) éteindra la machine
